**Status:** Done

## Story

**Como** usuário macOS  
**Eu quero** organizar minhas transcrições com favoritos, tags e preferências persistidas  
**Para que** eu personalize o dashboard e recupere rapidamente os itens mais importantes  

---

## Acceptance Criteria

- [x] Favoritos deixam de depender apenas de `localStorage` e passam a persistir via `audioflow_design.db`
- [x] O frontend consome `GET/POST/DELETE /api/favorites/:id` para marcar e listar favoritos
- [x] O frontend oferece fluxo mínimo para criar tags, listar tags e associá-las a transcrições via API
- [x] Preferências visuais e de interação do dashboard persistem via `GET/PATCH /api/preferences`
- [x] O companion web mantém a separação entre `AudioFlow.sqlite` (read-only) e `audioflow_design.db` (writeable metadata)
