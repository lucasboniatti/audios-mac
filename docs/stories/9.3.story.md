# Story 9.3: Sincronizar transcricoes com reconciliacao segura de conflito

**Epic:** EPIC-007 (AudioFlow Web Sync)  
**Status:** Draft  
**Priority:** COULD  
**Effort:** L (Large)  
**Phase:** 9 - Cloud Sync & Hybrid Companion  

---

## Story

**Como** usuario do companion web com conta cloud  
**Eu quero** sincronizar minhas transcricoes entre a base local e a nuvem  
**Para que** eu mantenha continuidade de historico sem perder dados quando houver conflito ou indisponibilidade da cloud  

---

## Acceptance Criteria

- [ ] O modelo cloud de transcricoes inclui `user_id`, `local_id`, `version`, `sync_status` e `source_updated_at` com isolamento por RLS
- [ ] O bridge local consegue criar e atualizar registros cloud a partir do historico local sem escrever indevidamente no CoreData do app
- [ ] Divergencias de versao registram conflito em `sync_conflicts` e impedem sobrescrita silenciosa dos dados
- [ ] Falhas de sync mantem o companion local operacional e deixam o estado `pending`/`conflict` observavel para reconciliacao posterior
- [ ] As APIs cloud de listagem, busca e estatisticas usam client Supabase por requisicao com filtros validados e escopo por usuario

## Tasks / Subtasks

- [ ] Implementar o contrato executavel de `transcriptions` e `sync_conflicts` definido na arquitetura revisada
- [ ] Adaptar o bridge Node para sincronizar registros locais com versionamento, `sync_status` e reconciliacao explicita
- [ ] Expor nas APIs do companion os estados `synced`, `pending` e `conflict` sem esconder falhas de sincronizacao
- [ ] Garantir que busca, listagem e stats do modo cloud usem filtros seguros e client Supabase por requisicao
- [ ] Cobrir cenarios de conflito, indisponibilidade de cloud e recuperacao posterior sem quebrar o modo local
