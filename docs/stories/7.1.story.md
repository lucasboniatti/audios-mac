# Story 7.1: Disponibilizar companion web local autenticado

**Epic:** EPIC-002 (AudioFlow Web Companion)  
**Status:** Done

## Story

**Como** usuário macOS  
**Eu quero** acessar o companion web local com login por sessão  
**Para que** eu visualize minhas transcrições no navegador sem expor páginas e APIs protegidas  

---

## Acceptance Criteria

- [ ] `POST /auth/login` aceita a senha local configurada, retorna `token` e `username`, e o cliente persiste a sessão
- [ ] `GET /auth/verify` valida a sessão atual e páginas protegidas redirecionam para `login.html` quando o token não existe ou expira
- [ ] `POST /auth/logout` invalida a sessão e o frontend remove os dados locais de autenticação
- [ ] `AuthAPI.fetchAPI()` anexa o header `Authorization: Bearer <token>` e trata `401` com redirect consistente
- [ ] `login.html` trata erro de senha incorreta, submit por Enter e redireciona para `/` quando a sessão já está válida
