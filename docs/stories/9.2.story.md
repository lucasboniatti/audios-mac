# Story 9.2: Adicionar perfil cloud com avatar e preferencias remotas

**Epic:** EPIC-007 (AudioFlow Web Sync)  
**Status:** Draft  
**Priority:** COULD  
**Effort:** M (Medium)  
**Phase:** 9 - Cloud Sync & Hybrid Companion  

---

## Story

**Como** usuario autenticado do companion web  
**Eu quero** manter meu perfil, avatar e preferencias remotas  
**Para que** minha experiencia cloud seja personalizada sem quebrar as preferencias locais existentes  

---

## Acceptance Criteria

- [ ] O companion cloud consegue ler e atualizar o perfil do usuario autenticado via `GET/PATCH /api/profile`
- [ ] O usuario consegue enviar, trocar e remover o avatar usando o bucket `avatars` com isolamento por pasta de usuario
- [ ] As preferencias remotas do perfil coexistem com o fallback local atual sem perder `theme_mode`, `accent_color`, `auto_paste_enabled` e `sound_feedback_enabled`
- [ ] Alteracoes de nome, avatar ou preferencias refletem na interface sem exigir novo login
- [ ] Acesso a perfil e storage respeita RLS e nao permite leitura ou escrita de dados de outro usuario

### Restrições

- O perfil cloud nao substitui o fallback local; ele convive com ele.

- Campos de preferencias remotas precisam continuar coerentes com o contrato local ja usado por [theme.js](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/theme.js).
