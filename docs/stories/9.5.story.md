# Story 9.5: Sincronizar tags e favoritos no modo cloud

**Epic:** EPIC-007 (AudioFlow Web Sync)  
**Status:** Draft  
**Priority:** COULD  
**Effort:** M (Medium)  
**Phase:** 9 - Cloud Sync & Hybrid Companion  

---

## Story

**Como** usuario autenticado do companion web  
**Eu quero** sincronizar meus favoritos e tags no modo cloud  
**Para que** minha organizacao pessoal acompanhe meu historico sem perder o fallback local ja existente  

---

## Acceptance Criteria

- [ ] Favoritos e tags do modo cloud ficam isolados por usuario autenticado e respeitam RLS
- [ ] O companion consegue listar, criar e associar tags no modo cloud usando IDs validados e escopo por usuario
- [ ] Favoritos cloud e tags cloud podem ser reconciliados com o metadata local sem apagar silenciosamente o estado existente
- [ ] [favorites.html](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/favorites.html) e a visao principal mostram metadata cloud quando disponivel e degradam para o fallback local quando necessario
- [ ] Conflitos de metadata ligados a transcricoes divergentes nao usam sobrescrita silenciosa; a reconciliacao segue o contrato de versionamento do `EPIC-007`

## Tasks / Subtasks

- [ ] Adaptar os endpoints de favoritos e tags para operar em modo cloud autenticado sem vazar dados entre usuarios
- [ ] Integrar reconciliacao de favoritos/tags ao contrato de sync de transcricoes e conflito da story `9.3`
- [ ] Atualizar [favorites.html](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/favorites.html), [index.html](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/index.html) e [auth.js](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/auth.js) para alternar entre metadata local e cloud
- [ ] Cobrir cenarios de tag duplicada, favorito duplicado, conflito de metadata e fallback offline

## Dev Notes

### Contexto Atual

- O modo local entregue na story [7.3.story.md](/Users/lucasboniatti/Desktop/Projetos/audios-mac/docs/stories/7.3.story.md) usa SQLite separado para `favorites`, `tags` e `transcription_tags`, conforme [ARCHITECTURE.md](/Users/lucasboniatti/Desktop/Projetos/audios-mac/docs/architecture/ARCHITECTURE.md#L246).
- Os endpoints locais atuais para preferencias, favoritos e tags estao em [server.js](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/server.js#L507) e os helpers de consumo estao em [auth.js](/Users/lucasboniatti/Desktop/Projetos/audios-mac/frontend/auth.js#L216).
- A arquitetura cloud do `EPIC-007` define `public.tags`, endpoints de tags e metadata de favoritos/tags nas transcricoes em [audioflow-web-supabase-architecture.md](/Users/lucasboniatti/Desktop/Projetos/audios-mac/docs/architecture/audioflow-web-supabase-architecture.md#L215) e [audioflow-web-supabase-architecture.md](/Users/lucasboniatti/Desktop/Projetos/audios-mac/docs/architecture/audioflow-web-supabase-architecture.md#L450).
- Existe uma diferenca estrutural importante entre IDs locais inteiros (`Z_PK`/SQLite) e IDs cloud UUID, e essa reconciliacao precisa ficar explicita na implementacao.

### Dependencies

- Depende da autenticacao cloud de `9.1`.
- Depende do contrato de sync seguro da story `9.3`.

### Restrições

- O modo local com `audioflow_design.db` continua sendo fallback suportado e nao pode ser apagado implicitamente.
- Tags e favoritos cloud precisam ficar no escopo do usuario autenticado; nao basta replicar o schema local atual sem isolamento.
- Reconciliacao de metadata nao pode violar o contrato de versionamento e conflito ja definido para transcricoes.
